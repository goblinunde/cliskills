# Brainstorming

Do design work before implementation. No code, scaffolding, or implementation skills until the design is presented and approved.

Required flow:
1. Explore project context.
2. Offer the visual companion separately if the discussion will benefit from visuals.
3. Ask one clarifying question at a time.
4. Propose 2-3 approaches with tradeoffs and a recommendation.
5. Present the design in sections and get approval.
6. Write the spec to `docs/superpowers/specs/YYYY-MM-DD--design.md`.
7. Self-review the spec for ambiguity, gaps, and contradictions.
8. Ask the user to review the written spec.
9. Hand off to `superpowers:writing-plans`.

Rules:
- Every project goes through this process, even simple ones.
- Decompose oversized requests into smaller subprojects before writing a spec.
- Follow existing codebase patterns when designing changes.
