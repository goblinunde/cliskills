# Requesting Code Review

Request code review early and often so issues are caught before they compound.

Typical triggers:
- After each task in subagent-driven development.
- After a major feature.
- Before merging to the main branch.
- When stuck and needing a fresh perspective.

Workflow:
1. Determine the review range with a base SHA and head SHA.
2. Dispatch a focused reviewer with the requirements or plan and a short summary of what changed.
3. Act on the results by severity.

Rules:
- Fix critical issues immediately.
- Fix important issues before proceeding.
- Do not skip review because the change seems simple.
